{Number, enumerated from A3.Constants.BLEND_TYPES} blendType

The mesh's render type. This can be either normal or additive.

{Object} buffers

The buffers to be used for this mesh

{Number} colorDataSize

Sets the size of each vertex's color value, typically 3 (RGB)

{Boolean} depthTest

Declares whether or not the object is to be depth tested. By default if the object is transparent we should disable the depth test, but there may be instances where this is not cool, and so you have the option

{Boolean} dynamic

Determines if we store the array of vertices, colours, etc once we have bound to the buffers. Also used for the STATIC_DRAW / DYNAMIC_DRAW setting in the buffer. Only set to true if you plan to update the buffer values

{A3.Core.Objects.Geometric.Geometry} geometry

The geometry abstraction. Useful to keep around if you update the underlying data in any way.

{A3.Core.Math.Matrix3} matrixNormals

The matrix for the normals. Used to convert the normal into eye space so that we can do lighting calculations more easily

{Number} normalDataSize

Sets the size of each vertex's normal, typically 3

{Number} opacity

The mesh's opacity. Typically a value between 0 and 1.

{Number} renderType

The mesh's render "mode" - either particles or a solid mesh

{Shader} shader

The shader to use for this mesh

{Boolean} transparent

Declares whether or not the object is transparent. If the opacity is set to anything other than 1 or the mesh is set to be transparent (via the constructor: data.transparent) then it will be true. By default, it will be false.

{Number} uvDataSize

Sets the size of each vertex's UV coordinate, typically 2 (x,y)

{Number} vertexDataSize

Sets the size of each vertex's data, which will normally be 3 (x,y,z)
